Yapily Documentation

As part of my responsibilities as Implementation Engineer and Implementation Engineer - Team Lead at Yapily, I look after our documentation and try to keep it up to date. We interact with the documentation via a private repository in Bitbucket, where we push our local changes and create PRs to be reviewed. At the moment my responsibility is to review my team's PRs, do bigger changes, and coordinate and mantain processes with other teams to ensure the content is accurate and up to date.

Our documentation is mostly Markdown, but sometimes some HTML might be used where Markdown falls short. The style sheets are already written, so we only use CSS for minor changes or fixes. We work localy so there is also basic Command Line and Git usage.
